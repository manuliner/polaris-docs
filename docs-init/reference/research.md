# Research workflow for documentation

## Goals

Collect authoritative sources **before** editing architecture or runbooks.

## Steps

1. Identify technology names and versions from the repo (package manifests, Docker images).
2. Prefer upstream docs (`ctx7` / internal doc MCP) over uncached general knowledge for API details.
3. Record citations (URL or library ID) at the bottom of the research note; avoid pasting huge dumps into skills.
4. Promote stable findings into `docs/` with MOC links; discard scratch notes.

## Output

Produce a compact bullet list of decisions and links, not raw transcript.
